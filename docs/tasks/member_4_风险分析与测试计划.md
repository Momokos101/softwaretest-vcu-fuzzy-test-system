**第 4 章：风险评估方法**

详细说明 5 维度评分算法，写清楚**为什么**这么设计（这是考官会问的）：

```markdown
## 评分维度设计依据（参考 ISTQB Risk-Based Testing 原则）

**Criticality（35% 权重）**
权重最大，因为一个需求的业务重要性决定了它出问题时的影响范围。
- Safety（10分）：直接影响安全检测，出错后果最严重
- State Transition（8分）：影响系统状态切换，bug 难以复现
- State Control（7分）：控制关键标志位
- Input Validation（6分）：验证输入有效性
- Timing（4分）：时序约束，不直接影响安全

**Boundary Sensitivity（25% 权重）**
有精确数值边界的需求更容易因边界处理错误产生 bug。
- 有精确边界（4.8V / 7.7V / 12.0V）= 10分
- 有范围无精确值 = 6分
- 无边界 = 2分

**Complexity（20% 权重）**
条件越多，测试组合越复杂，遗漏的可能性越大。
计算：条件数 × 2 + 范围数 × 2（上限10）

**State Impact（15% 权重）**
影响多个系统状态的需求，单个 bug 会引发连锁反应。

**Testability（5% 权重）**
直接可测的需求优先级可相对降低，难以测试的需求反而风险更高。
```

**完整的 7 套测试套件设计：**

| TS-04 | 多信号条件组合 | **Decision Table** | REQ-001~009 | High | 真实测试数据证明：任一信号异常导致整体FAIL，需验证组合规律 |
| TS-05 | READY标志一致性 | **EP** | REQ-009 | Medium | vehicle_state=170时ready_flag必须=1，需单独验证一致性 |
| TS-06 | CC2 GAN 序列测试 | **GAN-based** | REQ-001, REQ-004 | Medium | 补充EP/BVA覆盖不到的连续变化过程（电压缓慢越界场景） |

**为每套写清楚"为什么选这个技术"（评委会问）：**
- TS-01 选 BVA 而不是 EP：因为 CC2 边界点（4.8V、7.7V）是真实数据发现的 PASS/FAIL 分界，边界测试比等价类中值测试更有价值
- TS-04 选决策表：因为需要验证多信号同时存在时的组合效果，单信号测试覆盖不到
